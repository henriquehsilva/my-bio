Roteiro de peer-evaluation

Esta versão é diferente da clássica: além de contar operações, você valida 4 estratégias, métrica de disorder, seletores em runtime, modo --bench e o checker. Siga na ordem. Copie cada comando direto no terminal.

00

Setup — helpers

Cole no topo do terminal antes de tudo. Poupa digitação no resto do roteiro.

bash · helpers
# gera N inteiros aleatorios SEM duplicatas (com negativos)
gen() { python3 -c "import random,sys; n=int(sys.argv[1]); print(' '.join(map(str, random.sample(range(-2147483648,2147483647), n))))" "$1"; }

# atalho de correcao: roda push_swap e valida com o checker
chk() { ARG="$1"; ./push_swap $ARG | ./checker $ARG; }

02

Compilação

O make && make é o teste decisivo de relink.

bash
make && make        # 2a vez NAO pode relinkar -> "Nothing to be done"
make clean && make fclean && make re
make bonus          # mandatory e bonus avaliados separados
norminette          # erro de norma = 0, inclusive nos *_bonus
03

Tratamento de erro

Error\n vai no stderr — nunca no stdout.

bash
./push_swap                  # nada, volta pro prompt, exit 0
./push_swap ""               # Error
./push_swap "  "             # Error
./push_swap 1 2 2 3          # duplicata -> Error
./push_swap 1 2 abc          # nao-inteiro -> Error
./push_swap 2147483648       # > INT_MAX -> Error
./push_swap -2147483649      # < INT_MIN -> Error
./push_swap 1 2 3 2>/dev/null  # Error NAO pode estar no stdout
04

Casos-base e pequenos

A saída são só operações por \n, nada mais.

bash
./push_swap 0                 # nada (ja ordenado)
./push_swap 2 1               # sa
./push_swap 1 2 3             # ja ordenado -> nada
./push_swap 5 1 4 3 2 | ./checker 5 1 4 3 2   # OK
./push_swap 3 2 1 | cat -A    # confere: sem espacos/linhas extras
05

Seletores de estratégia

O coração da v1.1. Valide TODAS com o checker, não só conte ops.

bash
ARG=$(gen 100)
for s in --simple --medium --complex --adaptive; do
  echo -n "$s -> "; ./push_swap $s $ARG | ./checker $ARG
done
./push_swap $ARG | ./checker $ARG    # sem flag = adaptive (default)
06

Disorder e thresholds do adaptive

Deve trocar de método em 0.2 e 0.5.

bash
# quase ordenado -> disorder baixo -> O(n^2)
./push_swap --bench $(seq 1 100 | tr '\n' ' ') 2>&1 >/dev/null
# invertido -> disorder ~1 -> O(n log n)
./push_swap --bench $(seq 100 -1 1 | tr '\n' ' ') 2>&1 >/dev/null
07

Modo --bench e o teste do pipe

Bench no stderr, ops no stdout. Erro nº1 das duplas.

bash
ARG=$(gen 100)
./push_swap --bench $ARG 2> bench.txt | ./checker $ARG   # tem que dar OK limpo
cat bench.txt
08

Performance

Pior caso sobre muitas execuções, no modo default (adaptive).

bash · pior caso
# 100 numeros: pior caso em 200 execucoes
worst=0; for i in $(seq 200); do n=$(./push_swap $(gen 100) | wc -l); ((n>worst)) && worst=$n; done; echo "max100=$worst"
# 500 numeros
worst=0; for i in $(seq 200); do n=$(./push_swap $(gen 500) | wc -l); ((n>worst)) && worst=$n; done; echo "max500=$worst"
bash · correção em massa
for i in $(seq 200); do ARG=$(gen 100); r=$(./push_swap $ARG | ./checker $ARG); [ "$r" != "OK" ] && echo "FALHOU: $ARG"; done; echo "fim"
09

Memória e crash

Zero leaks em todos os caminhos: sucesso, erro e sem-args.

bash
ARG=$(gen 100)
valgrind --leak-check=full --show-leak-kinds=all ./push_swap $ARG >/dev/null
valgrind ./push_swap 1 2 abc        # erro tambem nao pode vazar
valgrind ./push_swap                # sem args tambem
./push_swap $(gen 500) >/dev/null; echo $?   # sem segfault em entrada grande
10

Bonus — checker

Só avaliado se o mandatory estiver perfeito.

bash
ARG="3 2 1"
./push_swap $ARG | ./checker $ARG           # OK
echo -e "sa\npb\nlol" | ./checker $ARG      # Error (instrucao inexistente) no stderr
./checker                                    # sem arg -> nada
echo "" | ./checker 3 2 1                    # KO (nao ordenou)
echo -e "pb\npb\npb" | ./checker 3 2 1       # KO (b nao vazia no fim)
