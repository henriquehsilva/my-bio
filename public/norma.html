The Norm (42) v4.1 — guia rápido com exemplos

Este documento resume as regras mais comuns da Norm e mostra exemplos do que quebra a norma versus o que passa. Baseado no PDF oficial “The Norm — Version 4.1”.

1) Naming (nomes)

  • Tipos devem seguir prefixos: s_ (struct), t_ (typedef), u_ (union), e_ (enum), g_ (global). :contentReference[oaicite:2]{index=2}
  • Identificadores: apenas minúsculas, dígitos e _ (snake_case). Sem maiúsculas. :contentReference[oaicite:3]{index=3}
  • Nomes de arquivos e pastas: apenas snake_case. :contentReference[oaicite:4]{index=4}
  • ASCII apenas (exceto dentro de strings/chars literais). :contentReference[oaicite:5]{index=5}
  • Globais não const ou não static são proibidas (salvo exceções do projeto). :contentReference[oaicite:6]{index=6}
Erro — nomes fora do padrão
typedef struct Pessoa
{
    char *Nome;
} Pessoa;        // typedef sem t_, struct sem s_, campos com maiúsculas
int Global;      // global sem g_, e nem const/static
Ok — nomes conforme a Norm
typedef struct s_person
{
    char    *name;
}   t_person;

static int  g_counter; /* global ok: g_ + static */

Dica prática: use nomes autoexplicativos em inglês e separados por underscore. :contentReference[oaicite:7]{index=7}

2) Formatting (formatação)

  • Função: no máximo 25 linhas (não contando as chaves da função). :contentReference[oaicite:8]{index=8}
  • Linhas: no máximo 80 colunas (inclui comentários). Tabs contam como espaços. :contentReference[oaicite:9]{index=9}
  • Indentação: usar TAB real de 4 colunas (não 4 espaços). :contentReference[oaicite:10]{index=10}
  • Chaves sozinhas em sua própria linha (exceto declaração de struct/enum/union). :contentReference[oaicite:11]{index=11}
  • Declarações no início da função; 1 variável por linha; sem init junto com declaração (exceto global/static/const). :contentReference[oaicite:12]{index=12}
  • Sem duas linhas vazias seguidas; linha vazia não pode conter espaços/tabs. :contentReference[oaicite:13]{index=13}
  • Operadores e operandos com exatamente 1 espaço; vírgula/; seguidos de espaço (salvo fim de linha). :contentReference[oaicite:14]{index=14}
  • Asterisco do ponteiro “gruda” no nome da variável (ex.: char *s é errado, char *s?? → na Norm é char *s “* junto do nome”: char *s vs char *s; o correto é char *s com *s colado no identificador: char *s → char *s (ex.: char *str deve virar char *str? Na Norm: char *str não; é char *str e * colado em str: *str). :contentReference[oaicite:15]{index=15}
  • Estruturas de controle usam chaves, exceto quando existe 1 instrução em 1 linha. :contentReference[oaicite:16]{index=16}
Erro — 2 instruções na mesma linha + espaços ruins 
if (x>0) { y=1; z=2; }   // 2 instruções na linha, espaços errados, chaves na mesma linha
Ok — 1 instrução por linha + espaçamento + chaves
if (x > 0)
{
	y = 1;
	z = 2;
}

Importante: configure seu editor pra inserir TAB real. A Norm valida isso. :contentReference[oaicite:17]{index=17}

3) Functions (funções)

  • No máximo 4 parâmetros nomeados. :contentReference[oaicite:18]{index=18}
  • Sem argumentos → protótipo deve usar void. :contentReference[oaicite:19]{index=19}
  • No máximo 5 variáveis por função. :contentReference[oaicite:20]{index=20}
  • return deve estar entre parênteses (exceto retorno vazio). :contentReference[oaicite:21]{index=21}
  • Um TAB entre tipo de retorno e nome da função. :contentReference[oaicite:22]{index=22}
Erro — parâmetros demais + return sem parênteses
int sum(int a, int b, int c, int d, int e)
{
	return a + b; // return sem parênteses, params>4
}
Ok — até 4 params + return com parênteses
int	sum(int a, int b)
{
	return (a + b);
}

int	main(void)
{
	int	result;

	result = sum(2, 3);
	return (result);
}

4) struct / typedef / enum / union

  • Espaço após struct/enum/union no nome. :contentReference[oaicite:23]{index=23}
  • Não pode declarar struct em arquivo .c (declare no .h). :contentReference[oaicite:24]{index=24}
Erro — struct declarada no .c
/* file: foo.c */
struct s_point
{
	int	x;
	int	y;
};
Ok — struct no header + typedef t_
/* file: point.h */
#ifndef POINT_H
# define POINT_H

typedef struct s_point
{
	int	x;
	int	y;
}	t_point;

#endif

5) Headers (.h)

  • Header deve conter apenas: includes, declarações, defines, protótipos e macros. :contentReference[oaicite:25]{index=25}
  • Includes no início do arquivo. :contentReference[oaicite:26]{index=26}
  • Proibido incluir arquivo .c. :contentReference[oaicite:27]{index=27}
  • Usar header guard: se for ft_foo.h, guard deve ser FT_FOO_H. :contentReference[oaicite:28]{index=28}
  • Não incluir headers não usados (regra marcada com *: pode falhar em avaliação). :contentReference[oaicite:29]{index=29}
Erro — sem guard + inclui .c
/* bad_header.h */
# include "foo.c"   // proibido
int	sum(int a, int b);
Ok — guard + includes no topo
/* ft_math.h */
#ifndef FT_MATH_H
# define FT_MATH_H

int	sum(int a, int b);

#endif

6) 42 Header (cabeçalho padrão)

  • Todo arquivo .c e .h deve começar imediatamente com o header padrão da 42. :contentReference[oaicite:30]{index=30}
  • Deve conter infos atualizadas (login, email, datas). Regra marcada com *: pode reprovar em review. :contentReference[oaicite:31]{index=31}
Ok — exemplo de header (modelo)
/* ************************************************************************** */
/*                                                                            */
/*                                                        :::      ::::::::   */
/*   ft_math.c                                           :+:      :+:    :+:   */
/*                                                    +:+ +:+         +:+     */
/*   By: login <login@student.42...>                +#+  +:+       +#+        */
/*                                                +#+#+#+#+#+   +#+           */
/*   Created: 2026/01/11 00:00:00 by login              #+#    #+#             */
/*   Updated: 2026/01/11 00:00:00 by login             ###   ########.fr       */
/*                                                                            */
/* ************************************************************************** */

7) Macros e Preprocessor

  • #define criado por você deve ser usado só para valores literais/constantes (*). :contentReference[oaicite:32]{index=32}
  • Proibido criar #define para burlar a norma ou ofuscar código (*). :contentReference[oaicite:33]{index=33}
  • Macros multi-linha são proibidas. :contentReference[oaicite:34]{index=34}
  • Nome de macro em UPPERCASE. :contentReference[oaicite:35]{index=35}
  • Diretivas fora do escopo global são proibidas. :contentReference[oaicite:36]{index=36}
Erro — macro multi-linha
#define DO_THING(x) \
	do { (x)++; } while (0)   /* macro multi-linha: proibido */
Ok — constante literal
#define BUFFER_SIZE 1024

8) Forbidden stuff (proibidos)

  • Proibido: for, do...while, switch/case, goto. :contentReference[oaicite:37]{index=37}
  • Proibido operador ternário ?:. :contentReference[oaicite:38]{index=38}
  • Proibido VLA (Variable Length Array). :contentReference[oaicite:39]{index=39}
  • Proibido “tipo implícito” em declarações. :contentReference[oaicite:40]{index=40}
Erro — for + ternário + VLA
int	main(int argc, char **argv)
{
	int		i;
	char	str[argc];         // VLA (proibido)
	i = argc > 5 ? 0 : 1;      // ternário (proibido)
	for (i = 0; i < argc; i++) // for (proibido)
		(void)argv[i];
	return (0);
}
Ok — while + sem ternário + sem VLA
int	main(int argc, char **argv)
{
	int	i;

	i = 0;
	while (i < argc)
	{
		(void)argv[i];
		i++;
	}
	return (0);
}

9) Comments (comentários)

  • Comentários não podem estar dentro do corpo da função. :contentReference[oaicite:41]{index=41}
  • Comentários devem estar no fim da linha, ou em linha própria. :contentReference[oaicite:42]{index=42}
  • Recomendado (*): comentários em inglês e úteis. :contentReference[oaicite:43]{index=43}
Erro — comentário dentro do corpo
int	sum(int a, int b)
{
	/* adds two numbers */  // proibido dentro da função
	return (a + b);
}
Ok — comentário fora, ou fim de linha
/* sum: add two integers and return the result */
int	sum(int a, int b)
{
	return (a + b); /* simple arithmetic */
}

10) Files

  • Proibido incluir .c dentro de .c. :contentReference[oaicite:44]{index=44}
  • No máximo 5 definições de função por arquivo .c. :contentReference[oaicite:45]{index=45}
Erro — inclui .c
#include "other.c"  // proibido
Ok — inclui header
#include "ft_math.h"

11) Makefile (checklist)

  • Regras obrigatórias: $(NAME), all, clean, fclean, re. :contentReference[oaicite:46]{index=46}
  • all deve ser a regra padrão (rodar ao digitar apenas make). :contentReference[oaicite:47]{index=47}
  • Não pode relinkar sem necessidade (senão é considerado não-funcional). :contentReference[oaicite:48]{index=48}
  • Sem *.c, *.o etc.: todos os arquivos devem ser listados explicitamente. :contentReference[oaicite:49]{index=49}
Ok — mini-exemplo (didático)
NAME = myprog

CC = cc
CFLAGS = -Wall -Wextra -Werror

SRCS = main.c ft_math.c
OBJS = $(SRCS:.c=.o)

all: $(NAME)

$(NAME): $(OBJS)
	$(CC) $(CFLAGS) $(OBJS) -o $(NAME)

clean:
	rm -f $(OBJS)

fclean: clean
	rm -f $(NAME)

re: fclean all

Nota: o PDF diz que Makefiles não são checados pela norminette, mas são checados na avaliação. :contentReference[oaicite:50]{index=50}

Exemplo final: “passa na norma” (arquivo .c pequeno)

Abaixo um exemplo bem simples, obedecendo: return com parênteses, no máximo 5 variáveis, sem comentários dentro da função, while ao invés de for, estilo de chaves e espaçamento. Ajuste a indentação para TAB real no seu editor.

Ok — exemplo completo
/* ************************************************************************** */
/*                                                                            */
/*                                                        :::      ::::::::   */
/*   main.c                                             :+:      :+:    :+:   */
/*                                                    +:+ +:+         +:+     */
/*   By: login <login@student.42...>                +#+  +:+       +#+        */
/*                                                +#+#+#+#+#+   +#+           */
/*   Created: 2026/01/11 00:00:00 by login              #+#    #+#             */
/*   Updated: 2026/01/11 00:00:00 by login             ###   ########.fr       */
/*                                                                            */
/* ************************************************************************** */

#include <unistd.h>

static int	ft_putstr(char *str)
{
	int	i;

	i = 0;
	while (str[i] != '\0')
	{
		write(1, &str[i], 1);
		i++;
	}
	return (i);
}

int	main(void)
{
	ft_putstr("Hello, Norm!\n");
	return (0);
}
